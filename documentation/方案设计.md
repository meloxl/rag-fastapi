# 恋爱知识 RAG 问答 — 方案设计

> 本文档描述系统架构、模块职责、数据流与关键设计决策，供后续扩展功能时给 AI 提供上下文。  
> 关联文档：[需求分析.md](./需求分析.md) | 运行说明：[../README.md](../README.md)

---

## 1. 设计目标

1. **简单可跑**：单仓库、FastAPI API、一套本地 RAG 链路，适合第一个 RAG 项目学习与演示。
2. **概念清晰**：与 Spring AI 的 DocumentReader / DocumentTransformer / DocumentWriter / Advisor 一一对应。
3. **改动可控**：新增能力时优先扩展对应模块，避免 `main.py` 膨胀。
4. **可验收**：问答响应携带 `sources`，便于核对检索是否命中正确篇章。
5. **双后端可切换**：通过 `RAG_PROVIDER` 在本地 Chroma RAG 与阿里云百炼云知识库 RAG 之间切换，保留本地学习链路，同时支持云知识库简化开发。
6. **百炼双调用模式**：`bailian` 后端默认走 **Retrieve API（2023-12-29）** 显式检索云知识库，再复用 OpenAI 兼容 LLM 生成；百炼应用 completion API 保留为备用路径。

## 3. 技术栈

| 层级 | 选型 | 版本策略 |
|------|------|----------|
| Web | FastAPI + Uvicorn | `requirements.txt` 最低版本约束 |
| RAG 编排 | LangChain Core / Community / Chroma / OpenAI / Ollama | 见 `requirements.txt` |
| 百炼云知识库 | `alibabacloud-bailian20231229` SDK | Retrieve API 版本 **2023-12-29** |
| 向量库 | Chroma（`langchain-chroma`）/ 百炼云知识库 | local 持久化目录 `chroma_db/`；bailian 由云端托管 |
| Embedding | OllamaEmbeddings / OpenAIEmbeddings | local 模式使用；bailian retrieve 模式由云端完成向量化 |
| LLM | ChatOpenAI（OpenAI 兼容 API） | local 与 bailian retrieve 模式均调用硅基流动等兼容 Chat API；bailian app 备用模式由百炼应用完成生成 |
| 文档加载 | TextLoader | UTF-8，支持 autodetect_encoding |
| 文本切分 | MarkdownHeaderTextSplitter | 按 `#` / `####` 标题切分 |

## 5. 模块设计（对标 Spring AI）

### 5.1 DocumentReader — `app/rag/reader.py`

**职责**：从 `docs/` 读取所有 `.md` 文件，输出 LangChain `Document` 列表。

**核心函数**：

| 函数 | 说明 |
|------|------|
| `list_markdown_files()` | 列出知识目录下所有 `.md` 文件名 |
| `load_documents()` | 加载文档并填充 metadata |
| `sanitize_upload_filename()` | 上传文件名安全校验 |

**Metadata 约定**：

| 字段 | 来源 | 示例 |
|------|------|------|
| `source` | 文件名 | `恋爱常见问题和回答 - 单身篇.md` |
| `category` | 文件名解析 | `单身` / `恋爱` / `已婚` |
| `file_path` | 绝对路径 | 用于调试 |

**category 解析规则**（`_parse_category`）：文件名含「单身」「恋爱」「已婚」则映射对应分类，否则用文件名 stem。

---

### 5.2 DocumentTransformer — `app/rag/transformer.py`

**职责**：将整篇 Markdown 切分为语义完整的 chunk。

**实现**：`MarkdownHeaderTextSplitter`

```python
HEADERS_TO_SPLIT_ON = [
    ("#", "section"),      # 篇章级
    ("####", "question"),  # 单条问答
]
```

**设计理由**：知识库每条问答以 `####` 为标题，按标题切分可保证「问题 + 回答 + 推荐课程」在同一块或相邻块内，优于固定 500 字符盲切。

**切分后 metadata**：继承文件级 metadata，并追加 header 解析字段（如 `question`）。

---

### 5.3 DocumentWriter — `app/rag/writer.py`

**职责**：向量化并写入 / 加载 Chroma。

| 函数 | 说明 |
|------|------|
| `build_vector_store(chunks)` | **全量重建**：删除旧 `chroma_db/` 目录后 `Chroma.from_documents` |
| `load_vector_store()` | 加载已有持久化库 |
| `is_kb_built()` | 判断向量库目录是否存在且非空 |

**向量化实现**：委托 `app/rag/embeddings.py` 的 `get_embeddings()`，按 `EMBED_PROVIDER` 选择后端（见 5.4）。

**重建策略**：每次 `build_kb` 全量覆盖，避免重复 build 产生重复 chunk。适合当前小规模静态知识库；文档量大时可改为增量 upsert。

### 5.4 Embedding 工厂 — `app/rag/embeddings.py`

**职责**：统一创建 LangChain `Embeddings`，便于扩展多种向量化来源。

| EMBED_PROVIDER | 实现类 | 典型场景 |
|----------------|--------|----------|
| `ollama` | `OllamaEmbeddings` | 本地 `nomic-embed-text` 等 |
| `openai_compatible` / `siliconflow` | `OpenAIEmbeddings` + 自定义 `base_url` | 硅基流动 `Qwen/Qwen3-Embedding-8B`、OpenAI、其他兼容 API |

**扩展新厂商**：若为 OpenAI 兼容 `/v1/embeddings`，只需配置 `OPENAI_COMPATIBLE_API_BASE` + `EMBED_MODEL_ID`；若非兼容，在 `embeddings.py` 增加新 provider 分支。

**注意**：切换 provider 或模型后必须重新 `build_kb`，旧向量维度/空间不兼容。

### 5.5 阿里云 Credentials 凭据链封装 — `app/rag/aliyun_credentials.py`

**职责**：封装阿里云 `alibabacloud-credentials` SDK 的默认凭据链机制，为百炼 Retrieve API 提供安全、灵活的鉴权能力，无需在代码中硬编码 AK/SK。

**核心函数**：

| 函数 | 说明 |
|------|------|
| `get_credential_client()` | 创建 Credentials 客户端实例（无参构造，自动从默认凭据链读取凭证） |
| `validate_ram_credential()` | 校验 RAM AccessKey 合法性（检查格式、非空、非 DashScope API Key） |
| `get_credential_status()` | 诊断用：返回凭据状态（包括可用状态、AccessKey ID 预览、是否使用 STS Token、错误信息等，不暴露 Secret） |

**Credentials 默认凭据链支持的凭证源（优先级从高到低）**：

1. **环境变量**：`ALIBABA_CLOUD_ACCESS_KEY_ID` + `ALIBABA_CLOUD_ACCESS_KEY_SECRET`
   - 本项目还支持别名 `AK` / `SK`（在 `config.py` 中同步到标准变量）
2. **阿里云 CLI 配置**：`~/.aliyun/config.json`
3. **实例元数据**：ECS/ACK 实例角色（生产环境推荐，无需配置 AK/SK）
4. **其他来源**：如阿里云 Credentials 文件等

**设计亮点**：
- 遵循阿里云官方最佳实践，不在业务代码中传递 AK/SK
- 提供 `validate_ram_credential()` 预检查，提前发现配置问题
- 提供 `get_credential_status()` 便于排查问题，且不暴露敏感信息
- 主动检测并拒绝使用 DashScope `sk-` 开头的 API Key（Retrieve API 需要 RAM AccessKey）

**与其他模块的关系**：
- `bailian_retrieve.py` 调用 `get_credential_client()` 获取凭证
- `diagnostics.py` 调用 `get_credential_status()` 用于诊断
- `config.py` 中 `_sync_ram_credentials_to_env()` 同步别名变量到标准环境变量

### 5.6 百炼云知识库 Retrieve 客户端 — `app/rag/bailian_retrieve.py`

**职责**：在 `RAG_PROVIDER=bailian` 且 `BAILIAN_CALL_MODE=retrieve`（默认）时，通过百炼 OpenAPI **Retrieve** 接口从云知识库「恋爱大师集合」检索文本切片。

**核心函数**：

| 函数 | 说明 |
|------|------|
| `retrieve_chunks(question, top_k)` | 调用 SDK `retrieve_with_options`，返回 `list[RetrievedChunk]` |
| `test_retrieve_connection(query)` | 诊断用：测试 Retrieve API 连通性 |

**API 规格**（官方版本 2023-12-29）：

- 请求：`POST /{WorkspaceId}/index/retrieve`
- 端点：`bailian.cn-beijing.aliyuncs.com`（可通过 `BAILIAN_ENDPOINT` 覆盖）
- 鉴权：**RAM AccessKey**（`AK`/`SK` 或 `ALIBABA_CLOUD_ACCESS_KEY_ID`/`SECRET`），与 DashScope `sk-` API Key 不同
- 权限：RAM 用户需 `AliyunBailianDataFullAccess`（含 `sfm:Retrieve`）且加入对应业务空间

**Retrieve 请求参数（程序侧）**：

| 参数 | 来源 | 说明 |
|------|------|------|
| `IndexId` | `BAILIAN_INDEX_ID` | 云知识库 ID（控制台知识库详情页获取） |
| `WorkspaceId` | `BAILIAN_WORKSPACE_ID` | 业务空间 ID |
| `Query` | 用户问题 | 检索 query |
| `DenseSimilarityTopK` | `RETRIEVE_K` | 向量检索 Top-K，默认 3 |
| `EnableReranking` | `BAILIAN_RETRIEVE_ENABLE_RERANKING` | 是否开启重排序，默认 true |

**返回解析**：从 `Data.Nodes[]` 提取 `Text`、`Score`、`Metadata`（JSON，含 `doc_name`、`title`、`hier_title` 等），映射为 `RetrievedChunk`。

### 5.7 百炼应用客户端（备用）— `app/rag/bailian.py`

**职责**：在 `BAILIAN_CALL_MODE=app` 时调用百炼应用 completion API，由百炼应用绑定云知识库并完成检索增强问答。因曾出现 `App.AccessDenied`（403），**不作为默认路径**，仅作备用。

**核心函数**：

| 函数 | 说明 |
|------|------|
| `ask_bailian(question)` | 校验百炼配置，调用 `/apps/{BAILIAN_APP_ID}/completion`，返回 `AskResponse` |
| `_extract_answer(payload)` | 从百炼响应中提取回答文本 |
| `_extract_sources(payload)` | 尽量从百炼响应中的引用/文档字段映射到 `SourceItem` |

**配置要求**：

| 配置 | 说明 |
|------|------|
| `BAILIAN_APP_ID` | 百炼应用 ID |
| `BAILIAN_WORKSPACE_ID` / `DASHSCOPE_WORKSPACE_ID` | 子业务空间 ID，写入 `X-DashScope-WorkSpace` 请求头 |
| `BAILIAN_API_KEY` / `DASHSCOPE_API_KEY` | DashScope API Key（`sk-` 开头） |
| `BAILIAN_API_BASE` | 默认 `https://dashscope.aliyuncs.com/api/v1` |

### 5.8 百炼诊断 — `app/rag/diagnostics.py`

**职责**：按 `BAILIAN_CALL_MODE` 检查当前百炼后端配置与连通性。

| 模式 | 检测内容 |
|------|----------|
| `retrieve`（默认） | AK/SK、WorkspaceId、IndexId、Retrieve 试调用 |
| `app`（备用） | API Key、App ID、completion API 试调用 |

可通过 `python -m app.rag.diagnostics`、`python diagnose_bailian.py` 或 `GET /diagnostics` 触发。

---

### 5.9 查询增强 — `app/rag/chain.py`

**职责**：根据 `RAG_PROVIDER` 与 `BAILIAN_CALL_MODE` 选择 RAG 后端，并统一封装为 `AskResponse`。

**local 流程**：

1. `load_vector_store()` 加载 Chroma
2. `as_retriever(search_kwargs={"k": RETRIEVE_K})` 默认 Top-3
3. `_format_context(docs)` 将检索结果格式化为带分类/来源/问题的上下文
4. `_generate_answer()`：`ChatPromptTemplate` + `ChatOpenAI` + `StrOutputParser`
5. 返回 `AskResponse(question, answer, sources)`

**bailian / retrieve 流程（默认）**：

1. 判断 `RAG_PROVIDER == "bailian"` 且 `BAILIAN_CALL_MODE == "retrieve"`
2. `bailian_retrieve.retrieve_chunks(question)` 从云知识库检索切片
3. `_generate_answer()` 复用与 local 相同的 Prompt 与 LLM（硅基流动等）
4. `_to_sources()` 从 `RetrievedChunk.metadata` 映射 `SourceItem`（含从文档名推断 category）

**bailian / app 流程（备用）**：

1. 判断 `BAILIAN_CALL_MODE == "app"`
2. 调用 `bailian.ask_bailian(question)`
3. 由百炼应用绑定的云知识库完成检索增强
4. 将百炼响应映射为 `AskResponse`

百炼 retrieve 模式下，本地 `docs/`、`chroma_db/`、embedding provider 不参与 `/ask`；生成仍走 `LLM_API_KEY` 配置的 OpenAI 兼容 Chat API。

**Prompt 设计要点**（system 消息）：

- 角色：专业恋爱关系顾问
- 约束：仅依据上下文；无则明确说明；不虚构链接

**sources 结构**（`SourceItem`）：

```json
{
  "category": "恋爱",
  "source": "恋爱常见问题和回答 - 恋爱篇.md",
  "preview": "争吵发生时，先冷静情绪..."
}
```

## 6. API 设计

| 方法 | 路径 | 请求 | 响应 | 说明 |
|------|------|------|------|------|
| GET | `/health` | — | `HealthResponse` | 存活探测 + 建库状态 + 当前 RAG 后端 |
| POST | `/build_kb` | — | `BuildKbResponse` | local 模式 ETL 全流程：读 → 切 → 写 |
| POST | `/ask` | `{"question": "..."}` | `AskResponse` | 根据 `RAG_PROVIDER` 走 local 或 bailian 问答 |
| POST | `/upload` | `multipart/form-data` file | `UploadResponse` | 保存 `.md` 到 `docs/` |
| GET | `/diagnostics` | — | `dict` | bailian 模式下检查 Retrieve 或 App 连通性 |

**错误码约定**：

| 场景 | HTTP | detail |
|------|------|--------|
| 无 markdown 文档 | 400 | 知识目录中没有 .md 文档 |
| local 模式未建库就提问 | 400 | 请先调用 POST /build_kb |
| bailian retrieve 缺少 AK/SK | 400 | 请设置 ALIBABA_CLOUD_ACCESS_KEY_ID/SECRET 或 AK/SK |
| bailian retrieve 缺少 IndexId | 400 | 请设置 BAILIAN_INDEX_ID |
| bailian app 缺少 API Key | 400 | 请设置 BAILIAN_API_KEY 或 DASHSCOPE_API_KEY |
| 百炼 Retrieve 调用失败 | 500 | 百炼 Retrieve 调用失败：... |
| 百炼应用调用失败（备用） | 403/500 | App.AccessDenied 或 HTTP 错误 |
| 上传非法文件 | 400 | 文件名/格式校验失败 |
| 模型 API 或百炼服务不可用 | 500/503 | 问答失败或无法连接生成模型/百炼服务 |

**Swagger**：启动后访问 `http://127.0.0.1:8000/docs`

## 8. 数据流时序

### 8.1 构建本地知识库（build_kb / local 模式）

```
docs/*.md
  → reader.load_documents()          # List[Document]
  → transformer.split_documents()    # List[Document] chunks
  → writer.build_vector_store()      # Chroma + persist chroma_db/
  → 返回 chunk_count
```

### 8.2 问答（ask / local 模式）

```
用户 question
  → main.py 检查 RAG_PROVIDER=local 且 chroma_db 已构建
  → chain.ask_question()
  → writer.load_vector_store()
  → retriever.invoke(question)       # Top-K chunks
  → chain: context + prompt + LLM
  → AskResponse(answer, sources)
```

### 8.3 问答（ask / bailian / retrieve 模式，默认）

```
用户 question
  → main.py 跳过本地 chroma_db 检查
  → chain.ask_question()
  → bailian_retrieve.retrieve_chunks(question)
  → SDK: POST /{WorkspaceId}/index/retrieve
  → Body: IndexId, Query, DenseSimilarityTopK, EnableReranking
  → 鉴权: RAM AccessKey（AK/SK）
  → 云知识库「恋爱大师集合」返回 Nodes[]
  → chain._generate_answer(): context + prompt + LLM（硅基流动等）
  → AskResponse(answer, sources)
```

### 8.4 问答（ask / bailian / app 模式，备用）

```
用户 question
  → main.py 跳过本地 chroma_db 检查
  → chain.ask_question()
  → bailian.ask_bailian(question)
  → POST {BAILIAN_API_BASE}/apps/{BAILIAN_APP_ID}/completion
  → Header: X-DashScope-WorkSpace={BAILIAN_WORKSPACE_ID}
  → 百炼应用绑定云知识库「恋爱大师集合」完成检索增强
  → AskResponse(answer, sources)
```

### 8.5 上传（upload / local 模式）

```
UploadFile (.md)
  → sanitize_upload_filename()
  → 写入 DOCS_DIR/{filename}
  → 提示用户手动 POST /build_kb
```

## 9. 关键设计决策记录（ADR 摘要）

| 决策 | 选项 | 最终选择 | 理由 |
|------|------|----------|------|
| 架构拆分 | 单文件 vs 四模块 | **四模块** `app/rag/*` | 对标 Spring AI，便于扩展 |
| 切分策略 | 固定字数 vs 标题切分 | **MarkdownHeaderTextSplitter** | 文档为 Q&A 结构 |
| 建库时机 | 启动自动 vs 手动 API | **手动 `/build_kb`** | 便于观察 ETL、控制 Ollama 调用 |
| 上传后建库 | 自动 vs 手动 | **手动重建** | 避免误覆盖；行为可预期 |
| LLM 接入 | 本地 Ollama vs OpenAI 兼容 API | **OpenAI 兼容 Chat API** | 复用硅基流动等兼容服务，配置简单 |
| RAG 后端 | 仅本地 vs 云知识库可切换 | **`RAG_PROVIDER=local/bailian`** | 保留本地学习链路，同时支持百炼云知识库简化开发 |
| 百炼集成方式（V1.1） | 直接低层检索 API vs 应用 API | **百炼应用 API** | 应用绑定云知识库，代码侧改动最小 |
| 百炼集成方式（V1.2） | Retrieve API vs 应用 API | **Retrieve API 默认 + 应用 API 备用** | 应用 API 出现 App.AccessDenied；Retrieve 显式可控检索与 sources |
| bailian 生成策略 | 百炼一体生成 vs 外接 LLM | **Retrieve 检索 + OpenAI 兼容 LLM** | 与 local 模式 Prompt 一致，生成模型可独立配置 |
| 问答接口 | GET vs POST | **POST `/ask`** | 避免中文 URL 编码问题 |
| 项目文档位置 | 放 `docs/` vs 独立目录 | **`documentation/`** | 避免被 RAG 索引 |

## 11. 扩展接入指南（给 AI 的修改地图）

扩展某能力时，优先修改下表对应文件，避免跨层耦合：

| 想加的能力 | 主要修改文件 | 注意事项 |
|------------|--------------|----------|
| 新文档格式（PDF） | `reader.py` | 扩展 loader 分支，metadata 保持一致 |
| 新切分策略 | `transformer.py` | 可组合多个 splitter |
| 换向量库（FAISS/PGVector） | `writer.py` | 保持 `build/load/is_built` 接口 |
| 换向量化模型/厂商 | `embeddings.py` + `.env` | 切换后重建 `chroma_db` |
| 改 Prompt / 加引用格式 | `chain.py` | local 与 bailian retrieve 共用 `_generate_answer` |
| 百炼 Retrieve 参数调优 | `bailian_retrieve.py` + `.env` | DenseSimilarityTopK、EnableReranking 等 |
| 百炼 App 引用解析 | `bailian.py` | 仅 `BAILIAN_CALL_MODE=app` 时生效 |
| **百炼鉴权方式调整** | `aliyun_credentials.py` | 当前使用 Credentials 默认凭据链，如需新增凭证源修改此模块 |
| 百炼连通性排查 | `diagnostics.py` / `diagnose_bailian.py` | 按 CALL_MODE 分流诊断，调用 `get_credential_status()` 检查凭证 |
| 新 API 端点 | `main.py` + `models.py` | 业务逻辑仍下沉到 `rag/` |
| 按 category 过滤 | `chain.py` retriever | 使用 Chroma metadata filter |
| 上传后自动建库 | `main.py` upload | 调用 `build_kb` 逻辑抽成 service 函数 |
| 配置项 | `config.py` + `.env.example` | 同步更新 README 和 documentation |
| 新云知识库平台 | 新增 `app/rag/*_client.py` + `chain.py` 分流 | 保持 `AskResponse` 响应结构一致 |

**推荐新增模块**（当功能变复杂时）：

```
app/rag/
├── retriever.py   # 混合检索、rerank
├── memory.py      # 会话历史
├── service.py     # build_kb / ask 编排，供 main 调用
├── bailian_retrieve.py  # 百炼云知识库 Retrieve（已实现）
├── bailian.py           # 百炼应用 completion（备用，已实现）
└── *_client.py    # 其他云知识库平台客户端
```

## 14. 常见问题排查（补充）

### 问题 1：`No module named 'alibabacloud_credentials'`
- **原因**：未安装依赖
- **解决**：运行 `pip install -r requirements.txt`

### 问题 2：诊断显示「AccessKey ID 格式异常（应以 LTAI 开头）」
- **原因**：使用了 DashScope API Key（`sk-` 开头）或 STS Token 而非 RAM AccessKey
- **解决**：
  - 获取正确的 RAM AccessKey（以 `LTAI` 开头）
  - 确保该 RAM 用户已授权 `AliyunBailianDataFullAccess` 权限
  - 在 `.env` 中配置 `ALIBABA_CLOUD_ACCESS_KEY_ID` 和 `ALIBABA_CLOUD_ACCESS_KEY_SECRET`

### 问题 3：`failed to update OAuth tokens in config file` 警告
- **原因**：本地阿里云 CLI 配置文件权限问题
- **解决**：这是警告不影响功能；若需消除，可检查 `~/.aliyun/config.json` 权限或暂时忽略
