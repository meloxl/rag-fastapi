# 百炼应用 HTTP 403 问题排查指南

## 问题现象

```
访问 /ask 接口返回 500 Internal Server Error
错误信息：{"detail":"问答失败：百炼应用调用失败：HTTP 403 {\"code\":\"App.AccessDenied\",\"message\":\"App access denied.\",...}"}
```

## 根本原因

HTTP 403 表示 **访问被拒绝**，说明请求被百炼服务拒绝，常见原因包括：

| 原因 | 症状 | 解决方案 |
|------|------|--------|
| **API Key 已失效/过期** | 之前可用，现在返回 403 | 登录阿里云百炼平台，刷新或重新生成 API Key |
| **API Key 与 App ID 不匹配** | 使用错误的 App ID | 确保 `BAILIAN_APP_ID` 与当前 `BAILIAN_API_KEY` 对应 |
| **应用被禁用** | 在百炼平台删除或禁用了应用 | 在阿里云百炼控制台检查应用状态 |
| **账户配额/余额不足** | 账户已用尽额度 | 检查阿里云账户余额和百炼应用配额 |
| **权限不足** | API Key 对应的账户权限更改 | 确认账户权限，必要时重新授权 |

## 快速诊断步骤

### 第 1 步：检查环境变量

确保 `.env` 文件中已正确设置：

```bash
# 查看当前配置
cat .env | grep BAILIAN
```

应该看到：
```
RAG_PROVIDER=bailian
BAILIAN_APP_ID=w6gsdtpq67
BAILIAN_WORKSPACE_ID=ws-kjehcatf1uzce0xp
BAILIAN_API_KEY=sk-xxxxxxxxxxxxxxxx
BAILIAN_API_BASE=https://dashscope.aliyuncs.com/api/v1
BAILIAN_TIMEOUT=60
```

**关键检查项：**
- [ ] `RAG_PROVIDER` 设置为 `bailian`
- [ ] `BAILIAN_APP_ID` 不为空
- [ ] 子业务空间应用已配置 `BAILIAN_WORKSPACE_ID`
- [ ] `BAILIAN_API_KEY` 不为空（以 `sk-` 开头）
- [ ] `BAILIAN_API_BASE` 正确（默认：`https://dashscope.aliyuncs.com/api/v1`）

### 第 2 步：运行诊断工具

在项目根目录执行：

```bash
python -m app.rag.diagnostics
```

输出示例（成功）：
```
============================================================
百炼应用诊断信息
============================================================
API Base URL: https://dashscope.aliyuncs.com/api/v1
API Key: sk-xxxxxxxx...xxxx
App ID: w6gsdtpq67
超时设置: 60s
连接测试: 成功

✅ 诊断结果：配置正常，连接成功
============================================================
```

输出示例（403 错误）：
```
============================================================
百炼应用诊断信息
============================================================
API Base URL: https://dashscope.aliyuncs.com/api/v1
API Key: sk-xxxxxxxx...xxxx
App ID: w6gsdtpq67
超时设置: 60s
连接测试: 失败 - HTTP 403

❌ 诊断结果：
【HTTP 403 - 访问被拒绝】
可能的原因：
1. API Key 已失效、过期或被撤销
2. API Key 与 App ID 不匹配
3. 应用在百炼平台被禁用
4. 账户配额已用尽或余额不足
5. API Key 权限不足
...
============================================================
```

### 第 3 步：验证百炼平台配置

登录 [阿里云百炼平台](https://bailian.aliyuncs.com/)：

1. **检查 API Key**
   - 进入「个人中心」→「API 密钥」
   - 查看当前使用的 API Key 是否**仍有效**（未过期、未被撤销）
   - 如果不确定，**删除旧 Key 并生成新的**
   - 复制新 Key 更新 `.env` 中的 `BAILIAN_API_KEY`

2. **检查应用配置**
   - 进入「应用列表」或「我的应用」
   - 确认 App ID `w6gsdtpq67` 的应用**存在且已启用**
   - 检查应用是否绑定了**云知识库**
   - 验证应用的访问权限是否**正常**

3. **检查账户状态**
   - 确认账户**余额充足**
   - 检查**配额限制**是否还有剩余
   - 查看是否有**欠费**情况

## 解决方案

### 方案 A：重新生成 API Key（最常见）

```bash
# 1. 登录阿里云百炼平台
# 2. 进入「个人中心」→「API 密钥」
# 3. 删除或禁用旧 Key
# 4. 点击「创建新的 API Key」
# 5. 复制新 Key，更新 .env：
BAILIAN_API_KEY=sk-你的新key

# 6. 重启应用
python -m uvicorn app.main:app --reload
```

### 方案 B：重新关联 App ID

```bash
# 1. 登录阿里云百炼平台
# 2. 进入「应用列表」
# 3. 确认当前 App ID 与 API Key 是否对应
# 4. 如需要，创建新应用或使用其他应用的 ID
# 5. 更新 .env：
BAILIAN_APP_ID=应用的正确ID
```

### 方案 C：切换到本地 RAG（如暂时无法修复）

临时方案，使用本地 Chroma 向量数据库：

```bash
# .env 中改为：
RAG_PROVIDER=local

# 然后调用 POST /build_kb 构建本地知识库
curl -X POST http://localhost:8000/build_kb
```

## 常见问题 (FAQ)

**Q: 我之前能用，现在突然返回 403，是什么原因？**

A: 最可能是 API Key 过期或被阿里云撤销。百炼的 API Key 有有效期限制，需要定期更新。

**Q: 如何确认 App ID 是否正确？**

A: 登录百炼平台，在「应用列表」中找到应用，点进去查看应用详情中的 App ID。确保与 `.env` 中的值完全一致。

**Q: 账户配额用尽后会返回 403 吗？**

A: 是的。如果账户达到配额限制，百炼会返回 403 Forbidden。需要升级套餐或等待下周期重置。

**Q: 我能在代码中捕获这个错误吗？**

A: 可以。已在 [`app/main.py`](app/main.py:78-84) 中添加了特殊处理，会返回更详细的错误信息。
