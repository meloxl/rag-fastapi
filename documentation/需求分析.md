## 1. 项目背景

这是团队的**第一个 RAG（检索增强生成）应用**，目标是在标准 RAG 流程基础上做**适度简化**，快速落地一个可演示、可扩展的知识库问答系统。第一阶段以本地 Markdown + Chroma 为主；V1.1 引入百炼应用 API；**V1.2 将百炼默认路径改为 Retrieve API + 外接 LLM**，用于对比「自建 RAG」与「云知识库托管 RAG」的开发成本、费用与隐私边界。

**业务场景**：基于恋爱主题知识库，提供 **AI 恋爱知识问答**服务，覆盖单身、恋爱中、已婚三类典型关系阶段。知识库既可以来自本地 `docs/` + `chroma_db/`，也可以来自阿里云百炼控制台中已创建并上传文档的云知识库“恋爱大师集合”。

**参考项目**：[LangChain-RAG-FastAPI-Service](https://github.com/RMA-MUN/LangChain-RAG-FastAPI-Service/tree/base-rag)（企业级 RAG 对话服务）。本项目**不照搬**其 Django 用户体系、MySQL 会话、前端等复杂能力，仅借鉴「Reader / Transformer / Writer / 检索链」的分层思路。

## 2. RAG 流程：标准版 vs 本项目简化版

| 标准 RAG 步骤 | 本项目简化步骤 | 说明 |
|---------------|----------------|------|
| 1. 文档收集和切割 | **1. 文档准备** | 知识以 Markdown 形式存放在 `docs/`，结构尽量规范 |
| 2. 向量转化和存储 | **2. 文档读取** + **3. 向量转化和存储** | 读取与切分分开实现，向量写入 Chroma |
| 3. 切片过滤和检索 | （合并进步骤 4） | 第一版使用向量 Top-K 检索，不做混合检索/重排序 |
| 4. 查询增强和关联 | **4. 查询增强** | `local` 模式：Retriever + 领域 Prompt + OpenAI 兼容 LLM；`bailian` 模式（默认）：百炼 Retrieve API 检索云知识库切片 + 同一套 Prompt/LLM 生成；`bailian` 模式（备用）：百炼应用 completion 一体完成 RAG |

### 2.1 双 RAG 后端模式

当前系统通过 `RAG_PROVIDER` 支持两种问答后端；`bailian` 后端再通过 `BAILIAN_CALL_MODE` 区分调用方式：

| RAG_PROVIDER | 知识库来源 | 检索/生成方式 | 适用场景 |
|--------------|------------|---------------|----------|
| `local` | 本地 `docs/` + `chroma_db/` | 本地 Chroma 检索，再调用 OpenAI 兼容 Chat API 生成 | 学习 RAG 流程、数据留在本地、可控性强 |
| `bailian` | 阿里云百炼云知识库「恋爱大师集合」 | 见下表 `BAILIAN_CALL_MODE` | 简化 ETL/向量库维护，文档由云平台托管 |

**`bailian` 子模式（`BAILIAN_CALL_MODE`）**：

| BAILIAN_CALL_MODE | 检索 | 生成 | 鉴权凭证 | 说明 |
|-------------------|------|------|----------|------|
| `retrieve`（**默认**） | 百炼 Retrieve API（2023-12-29） | OpenAI 兼容 Chat API（硅基流动等） | RAM AK/SK + `BAILIAN_INDEX_ID` | 显式云检索，sources 可控；规避应用 API 403 |
| `app`（备用） | 百炼应用内部完成 | 百炼应用 completion 返回 | DashScope `sk-` API Key + `BAILIAN_APP_ID` | 曾遇 `App.AccessDenied`，保留备用 |

`/build_kb`、`/upload` 仍服务 local 模式；百炼模式的文档上传、切分、向量化和索引维护在百炼控制台完成。

## 4. 知识库需求

### 4.1 本地内置文档

项目已内置 3 篇 Markdown，位于 `docs/`：

| 文件 | 分类 | 内容 |
|------|------|------|
| `恋爱常见问题和回答 - 单身篇.md` | 单身 | 魅力提升、社交结识、线上交友、焦虑克服、相亲评估等 |
| `恋爱常见问题和回答 - 恋爱篇.md` | 恋爱 | 争吵处理、浪漫惊喜、保持自我、未来规划、缺点沟通等 |
| `恋爱常见问题和回答 - 已婚篇.md` | 已婚 | 工作家庭平衡、婚后浪漫、矛盾化解等 |

### 4.2 文档格式约定

- 格式：**Markdown**
- 篇章标题：`# 恋爱常见问题和回答 - xxx篇`
- 单条问答：`#### 问题标题` + 正文回答 + 可选「推荐课程」链接
- 切分策略依赖上述标题层级，**新增文档应遵循同一结构**，以保证检索 chunk 语义完整

### 4.3 文档扩展方式

- 手动放入 `docs/` 目录
- 通过 API `POST /upload` 上传 `.md` 文件
- **上传或增删文档后，必须调用 `POST /build_kb` 重建向量库**

### 4.4 百炼云知识库

百炼模式使用阿里云百炼控制台中已创建的云知识库：

| 项 | 要求 |
|----|------|
| 云知识库名称 | `恋爱大师集合` |
| 文档准备 | 文档已在百炼控制台上传并完成索引 |
| 知识库 IndexId | `BAILIAN_INDEX_ID`（控制台知识库详情页获取，**与应用 App ID 可能不同**） |
| 业务空间 ID | `BAILIAN_WORKSPACE_ID`，当前为 `ws-kjehcatf1uzce0xp` |
| Retrieve 鉴权 | RAM AccessKey（`.env` 中 `AK`/`SK` 或标准环境变量名） |
| 生成模型 | `LLM_API_KEY` 配置的 OpenAI 兼容 Chat API（retrieve 默认模式） |
| 应用绑定（备用） | `BAILIAN_CALL_MODE=app` 时，百炼应用需绑定该云知识库；`BAILIAN_APP_ID` 当前为 `w6gsdtpq67` |

百炼 retrieve 模式下，程序通过 SDK 调用 `POST /{WorkspaceId}/index/retrieve` 检索切片，不再读取本地 `docs/`、不写入 `chroma_db/`、不执行本地 embedding；生成仍由配置的 LLM 服务完成。本地文档保留用于 `local` 模式学习和回退。

## 5. 功能需求

### 5.1 必须实现（V1 + V1.2）

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|----------|
| F1 | 健康检查 | `GET /health` | 返回服务状态、是否已建库、文档数、模型配置、当前 RAG 后端 |
| F2 | 构建本地知识库 | `POST /build_kb` | `local` 模式能力：读取 `docs/*.md`，切分、向量化，持久化到 `chroma_db/` |
| F3 | 恋爱问答 | `POST /ask` | 根据 `RAG_PROVIDER` 选择本地 Chroma 或百炼云知识库，返回 `answer` + `sources` |
| F4 | 文档上传 | `POST /upload` | `local` 模式能力：支持上传 `.md` 到 `docs/`，提示用户重建知识库 |
| F5 | 百炼诊断 | `GET /diagnostics` / `diagnose_bailian.py` | bailian 模式下按 `BAILIAN_CALL_MODE` 检测 Retrieve 或 App 连通性 |

### 5.2 问答行为要求

- 角色定位：**专业恋爱关系顾问**
- 仅依据检索到的知识库内容回答，**不得编造**
- 知识库无相关内容时，明确回复「知识库中未找到相关内容」
- 回答语气：专业、温和、实用

### 5.3 百炼模式行为要求

**retrieve 模式（`BAILIAN_CALL_MODE=retrieve`，默认）**：

- `RAG_PROVIDER=bailian` 时，`/ask` 先调用百炼 Retrieve API 从云知识库「恋爱大师集合」检索切片，再调用 OpenAI 兼容 LLM 生成回答。
- 不要求本地 `chroma_db/` 已构建，`/ask` 不应因本地未建库而返回 400。
- RAM AK/SK 或 `BAILIAN_INDEX_ID` 缺失时，应返回明确配置错误。
- `sources` 从 Retrieve 返回的 `Nodes[].Metadata` 映射（`doc_name`、`title` 等）；可从文档名推断 `category`（单身/恋爱/已婚）。

**app 模式（`BAILIAN_CALL_MODE=app`，备用）**：

- `/ask` 调用百炼应用 completion API；应用须在控制台绑定云知识库。
- `BAILIAN_API_KEY` 缺失时，应返回明确配置错误。
- `sources` 尽量从百炼响应引用字段映射；未返回时允许空数组。

**共用**：

- 问答角色、语气、不得编造等行为要求与 5.2 一致。
- 可通过 `GET /diagnostics` 或 `python diagnose_bailian.py` 排查连通性问题。

### 5.4 明确不做（V1/V1.1 范围外）

- 用户注册 / 登录 / 权限
- 多轮会话记忆
- PDF / Word 等非 Markdown 格式解析
- 混合检索（BM25 + 向量）、重排序
- 上传后自动重建向量库（当前为手动触发，降低意外覆盖风险）
- Web UI 前端

## 6. 非功能需求

| 类别 | 要求 |
|------|------|
| 部署 | 本地运行 FastAPI；`local` 模式依赖本地向量库和模型 API；`bailian` retrieve 模式依赖百炼 Retrieve API（AK/SK）+ LLM API Key；`bailian` app 模式依赖 DashScope API Key |
| 隐私 | `local` 模式知识库与向量库均存本地；`bailian` 模式文档和检索由云平台托管，需接受额外费用与隐私边界 |
| 可维护 | 模块按 Reader / Transformer / Writer / Chain 拆分，便于对标 Spring AI 概念扩展 |
| 可配置 | 模型名、路径、检索条数等通过环境变量或 `.env` 配置 |
| Python | **3.11**（`pyproject.toml` 约束 `>=3.11,<3.12`，与 FastAPI 默认环境一致） |

## 7. 技术约束（已确认）

| 决策项 | 确认结果 |
|--------|----------|
| Web 框架 | FastAPI |
| RAG 编排 | LangChain |
| 向量库 | `local` 模式使用 Chroma；`bailian` 模式使用百炼云知识库（Retrieve API） |
| 百炼集成 | SDK `alibabacloud-bailian20231229`，Retrieve API 版本 **2023-12-29**；默认 retrieve + 外接 LLM，app completion 备用 |
| Embedding | Ollama `nomic-embed-text` 或 OpenAI 兼容 API（local 模式）；bailian retrieve 由云端完成 |
| LLM | OpenAI 兼容 Chat API `Qwen/Qwen3-VL-8B-Instruct`（local 与 bailian retrieve 共用） |
| 目录结构 | `app/rag/` 多模块拆分（含 `bailian_retrieve.py`、`bailian.py`、`diagnostics.py`） |
| API | `health` + `build_kb` + `ask` + `upload` + `diagnostics`；通过 `RAG_PROVIDER` / `BAILIAN_CALL_MODE` 切换 |

## 8. 验收用例

| 序号 | 输入问题 | 期望 |
|------|----------|------|
| 1 | 如何在社交场合主动结识心仪异性？ | 回答贴合单身篇；`sources.category` 含「单身」 |
| 2 | 恋爱中如何有效处理双方的争吵？ | 回答贴合恋爱篇；`sources.category` 含「恋爱」 |
| 3 | 婚后如何平衡工作与家庭责任？ | 回答贴合已婚篇；`sources.category` 含「已婚」 |
| 4 | `RAG_PROVIDER=local` 且未建库时调用 `/ask` | 返回 400，提示先 `build_kb` |
| 5 | 上传新 `.md` 后不 `build_kb` 直接提问 | `local` 模式仍使用旧向量库（行为符合设计） |
| 6 | `RAG_PROVIDER=bailian` 且未构建本地库时调用 `/ask` | 不检查本地 Chroma，直接走百炼 Retrieve（默认）或 App |
| 7 | `RAG_PROVIDER=bailian`，retrieve 模式但未配置 AK/SK | 返回明确配置错误 |
| 8 | `RAG_PROVIDER=bailian`，app 模式但未配置 API Key | 返回明确配置错误，提示 `BAILIAN_API_KEY` |
| 9 | `python diagnose_bailian.py`（retrieve 模式） | 输出 Retrieve 连通性与切片数量 |
| 10 | 恋爱类问题（bailian retrieve） | `answer` 有内容；`sources` 含云知识库文档引用 |

## 10. 术语表

| 术语 | 含义 |
|------|------|
| RAG | Retrieval-Augmented Generation，检索增强生成 |
| Chunk | 文档切分后的文本块，向量检索的最小单元 |
| Embedding | 文本向量表示，用于语义相似度检索 |
| Ollama | 本地运行开源 LLM / Embedding 模型的工具 |
| Chroma | 开源向量数据库，支持本地持久化 |
| 百炼 | 阿里云大模型服务平台，提供应用编排和云知识库能力 |
| DashScope | 阿里云灵积/百炼 API 调用体系；`sk-` API Key 用于模型与应用 completion |
| Retrieve API | 百炼 OpenAPI（2023-12-29），`POST /{WorkspaceId}/index/retrieve`，从云知识库检索切片 |
| IndexId | 云知识库主键 ID，由 CreateIndex 或控制台知识库详情获取，对应 `BAILIAN_INDEX_ID` |
| RAM AccessKey | 阿里云访问密钥（AK/SK），Retrieve API 鉴权所需，与 DashScope API Key 不同 |
| BAILIAN_CALL_MODE | bailian 后端子模式：`retrieve`（默认）/ `app`（备用） |
| RAG_PROVIDER | 本项目用于切换问答后端的配置项，支持 `local` / `bailian` |
